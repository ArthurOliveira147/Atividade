 O gerenciamento de requisitos é um conjunto de técnicas para documentar, analisar, priorizar e concordar com os
requisitos, de modo que as equipes de engenharia sempre tenham requisitos atuais e aprovados.

 O objetivo do gerenciamento de requisitos é garantir que as metas de desenvolvimento de software e produtos sejam 
atingidas com sucesso. O gerenciamento de requisitos oferece uma maneira de evitar erros por meio do rastreamento 
das alterações nos requisitos.

 Os principais itens a serem definidos em um plano de gerenciamento de requisitos são a visão geral do projeto, o
processo de coleta de requisitos, funções e responsabilidades, ferramentas e rastreabilidade.

 Etapas do gerenciamento de requisitos

1_Rastrear requisitos para itens de trabalho;

2_Consultar as partes interessadas após a implementação sobre as mudanças necessárias nos requisitos;

3_Usar o gerenciamento de testes para verificar e validar os requisitos do sistema;

4_Avaliar o impacto das mudanças.

 Por que é necessário gerenciar requisitos?

 Ter um plano de gerenciamento de requisitos é essencial para o sucesso de um projeto
porque permite que as equipes de engenharia controlem o escopo e direcionem o ciclo
de vida de desenvolvimento do produto.

 O propósito do Gerenciamento de Requisitos é gerenciar os requisitos do produto e dos componentes do produto do
projeto e identificar inconsistências entre os requisitos, os planos do projeto e os produtos de trabalho do
projeto.

 Alguns dos benefícios do gerenciamento de requisitos incluem:
Menor custo de desenvolvimento em todo o ciclo de vida;
Menos defeitos;
Risco minimizado para produtos críticos para a segurança;
Entrega mais rápida;
Reusabilidade;
Rastreabilidade;
Requisitos ligados a casos de teste;
Gerenciamento de configuração global.

 As chaves para o gerenciamento eficiente de requisitos incluem manter uma instrução clara dos Requisitos, juntamente
com os atributos aplicáveis para cada tipo de requisito e a Rastreabilidade para outros requisitos e outros produtos
de trabalho do projeto. A coleta de requisitos pode parecer uma tarefa bem precisa.
